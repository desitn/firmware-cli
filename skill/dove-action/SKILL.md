# Dove Action（高风险操作）

用于烧录、编译、复位的操作，会修改设备状态，**必须先向用户确认**。

## 环境说明

- **可执行文件**: `dove.exe`
- **运行环境**: Windows OS
- **前置条件**: 插件已预安装，无需手动初始化环境
- **查询命令**: `build --list`、`flash --list`、`port list`、`port at` 查询类命令在 `dove-query` skill 中，可直接执行

---

## 需确认的命令

以下命令会修改设备或执行长时间任务，**必须先确认**：

| 命令 | 风险说明 | 确认要求 |
|------|----------|----------|
| `flash` | 烧录会覆盖设备固件 | 告知用户固件将被覆盖 |
| `build` | 可能触发长时间编译（数分钟） | 告知用户耗时预估 |
| `build --index` | 执行指定编译命令 | 告知用户具体命令 |
| `build --name` | 执行指定编译命令 | 告知用户具体命令 |
| `build-and-flash` | 编译+烧录组合操作 | 双重确认 |
| `port at -c "AT+CFUN=1,1"` | 设备复位命令 | 告知用户设备将重启 |

## 触发条件

当用户**明确要求执行操作**时触发：

**明确触发：**
- "烧录固件" / "下载固件" / "flash" / "烧录"
- "编译" / "编译工程" / "build"
- "编译并烧录" / "一键编译下载"
- "复位设备" / "重启模块" / "AT+CFUN=1,1"

**不应触发：**
- 用户仅讨论编译/烧录概念，无执行意图
- 用户仅要求"列出编译命令"或"查看固件列表"（应使用 dove-query）
- 用户仅要求"查询设备信息"（应使用 dove-query 的 ATI 命令）

```

风险说明: 设备将重启复位，通信连接会中断

```

## 命令参考

### 固件烧录（需确认）

```bash
# 烧录固件（自动查找最新）- 需确认
dove.exe flash

# 烧录指定固件 - 需确认
dove.exe flash "C:/path/to/firmware.bin"

# 跳过自动进入下载模式
dove.exe flash --skip-dl-mode
```

### 固件编译（需确认）

```bash
# 编译固件（执行 active 命令）- 需确认
dove.exe build

# 按索引执行命令 - 需确认
dove.exe build --index 1

# 按名称执行命令 - 需确认
dove.exe build --name install
```

### 编译并烧录（需确认）

```bash
# 一键编译并烧录 - 需确认（双重风险）
dove.exe build-and-flash
```

### 设备复位（需确认）

```bash
# 设备复位 - 需确认
# AT 命令自动使用配置中 tag: "AT" 的端口
dove.exe port at -c "AT+CFUN=1,1" --timeout 10000
```

> **说明**: AT 复位命令自动使用 AT 标签端口。查询类 AT 命令（如 ATI）在 dove-query 中可直接执行。

## 端口标签配置

端口标签用于 AI 自动识别端口用途：

| 标签 | 用途 | AI 行为 |
|------|------|----------|
| **AT** | AT 命令端口 | `port at` 自动使用 |
| **DBG** | 调试日志端口 | `port monitor` 自动使用 |
| **invalid** | 无效端口 | AI 不使用 |

配置示例（dove.json）：
```json
{
  "comPorts": [
    { "port": "COM3", "tag": "AT" },
    { "port": "COM4", "tag": "DBG" }
  ]
}
```

## 约束规则

1. **必须先确认**：执行以上命令前，必须展示确认模板并等待用户确认
2. **不自动执行**：收到用户请求后，先展示风险说明，不直接执行
3. **错误只展示**：工具返回错误时，只展示给用户，不自动修复
4. **禁止替代方案**：工具返回错误时，禁止自动尝试调用脚本代替

### 错误处理

| 错误信息 | 处理方式 |
|----------|----------|
| "未找到固件文件" | 提示用户检查配置或指定路径 |
| "未找到工作空间" | 提示用户缺少 `quectel_build` 目录 |
| "下载工具不存在" | 提示用户检查 tools 目录 |
| "未找到构建命令" | 提示用户配置 buildCommand |
| "AT port not found" | 提示用户配置 `tag: "AT"` 端口 |

## 注意事项

- **查询命令在 dove-query skill**：`flash --list`、`build --list`、`port list`、`config`、`port monitor`、查询类 AT 命令可直接执行无需确认
- 用户说"列出编译命令"时应使用 dove-query 的 `build --list`
- 用户说"查询设备信息"时应使用 dove-query 的 `port at -c "ATI"`
- `invalid` 标签端口 AI 不使用，如果用户指定 invalid 端口需提示
