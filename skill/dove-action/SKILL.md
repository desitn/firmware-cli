# Dove Action（高风险操作）

用于烧录、编译、复位的操作，会修改设备状态，**必须先向用户确认**。

## 环境说明

- **可执行文件**: `dove.exe`
- **运行环境**: Windows OS
- **前置条件**: 插件已预安装，无需手动初始化环境

---

## 需确认的命令

以下命令会修改设备或执行长时间任务，**必须先确认**：

| 命令 | 风险说明 | 确认要求 |
|------|----------|----------|
| `flash` | 烧录会覆盖设备固件 | 告知用户固件将被覆盖 |
| `build` | 可能触发长时间编译（数分钟） | 告知用户耗时预估 |
| `build-and-flash` | 编译+烧录组合操作 | 双重确认 |
| `at -c "AT+CFUN=1,1"` | 设备复位命令 | 告知用户设备将重启 |

---

## 触发条件

当用户**明确要求执行操作**时触发：

**明确触发：**
- "烧录固件" / "下载固件" / "flash" / "烧录"
- "编译" / "编译工程" / "build"
- "编译并烧录" / "一键编译下载"
- "复位设备" / "重启模块" / "AT+CFUN=1,1"

**不应触发：**
- 用户仅讨论编译/烧录概念，无执行意图

## 确认模板（必须使用）

执行高风险命令前，必须使用以下格式确认：

```
即将执行: dove.exe [命令]
风险说明: [具体风险描述]
是否继续? 请确认后执行。
```

### 确认示例

**烧录固件：**
```
即将执行: dove.exe flash
风险说明: 将烧录固件到设备，当前设备固件将被覆盖
是否继续? 请确认后执行。
```

**编译工程：**
```
即将执行: dove.exe build
风险说明: 将执行固件编译，可能需要数分钟时间
是否继续? 请确认后执行。
```

**设备复位：**
```
即将执行: dove.exe at -c "AT+CFUN=1,1"
风险说明: 设备将重启复位，通信连接会中断
是否继续? 请确认后执行。
```

## 命令参考

### 固件烧录
```bash
# 烧录固件（自动查找最新）- 需确认
dove.exe flash

# 烧录指定固件 - 需确认
dove.exe flash "C:/path/to/firmware.bin"

# 跳过自动进入下载模式
dove.exe flash --skip-dl-mode
```

### 固件编译
```bash
# 编译固件 - 需确认
dove.exe build

# 列出可用编译命令（低风险，可直接执行）
dove.exe build --list

# 按索引执行命令 - 需确认
dove.exe build --index 1
```

### 编译并烧录
```bash
# 一键编译并烧录 - 需确认（双重风险）
dove.exe build-and-flash
```

### 设备复位（AT 命令）
```bash
# 设备复位 - 需确认
dove.exe at -c "AT+CFUN=1,1" --timeout 10000
```

## 约束规则

1. **必须先确认**：执行以上命令前，必须展示确认模板并等待用户确认
2. **不自动执行**：收到用户请求后，先展示风险说明，不直接执行
3. **错误只展示**：工具返回错误时，只展示给用户，不自动修复
4. **错误只展示**：工具返回错误时，禁止自动尝试调用脚本代替
5. 
### 错误处理

| 错误信息 | 处理方式 |
|----------|----------|
| "未找到固件文件" | 提示用户检查配置或指定路径 |
| "未找到工作空间" | 提示用户缺少 `quectel_build` 目录 |
| "下载工具不存在" | 提示用户检查 tools 目录 |
| "未找到构建命令" | 提示用户配置 buildCommand |

---

## 执行环境检查

执行操作前，应确认：

1. **工作目录**: 是否在项目根目录（包含 `quectel_build` 或已配置 `firmwarePath`）
2. **工具存在**: `dove.exe` 是否存在
3. **设备连接**: flash 命令需确认设备已连接

若环境不满足，应提示用户即可，不要尝试强行执行其他处理。
